01 — Overview

What this workflow does

This project is a recruiter AI agent that automatically screens incoming resumes from Gmail, extracts and structures candidate data with AI, logs it into a duplicate-free Google Sheets database, and drafts a tailored reply — all without manual intervention.

02 — Build Log

Key steps & debugging stories

Getting this system to run reliably wasn't a straight line. Below are the key build steps in order, and the real errors I ran into along the way — with how I traced and fixed each one.

STEP 01
Smart Pre-AI Spam Filter
optimization

To avoid burning API calls on obvious spam, I added a keyword filter node that runs before the request ever reaches the AI Agent.

debug_log:

My own legitimate resume submission was flagged as a false negative. I traced it to a case-sensitivity issue — the filter matched lowercase apply, but my email used Application. Fixed by enabling Ignore Case on the filter condition.

recruitment-keywords.params
Enabling "ignore case" for reliable keyword matching.
STEP 02
Duplicate Prevention (Upsert Logic)
error resolved

To prevent duplicate rows when a candidate emails more than once, I switched the Google Sheets node from a plain "Append" to an Append or Update Row operation.

debug_log:

n8n threw 'columns.matchingColumns' is required — the node had no way to know which existing row to match against. Resolved by setting Contact as the unique matching key, so the workflow now checks for an existing candidate before inserting or updating.

db-save-update-candidate.error
The n8n error shown when no matching key was set.
STEP 03
Syntax Bug in Data Mapping
syntax fixed

While writing extracted data into the Skills field, n8n returned an invalid syntax error.

debug_log:

The expression was missing a closing brace: {{ JSON.parse($json.output).skills}. Adding the second }} fixed the parsing instantly — a small typo that took the whole branch down until I caught it.

skills-expression.field
The Skills expression field before the fix.
STEP 04
The Mystery Disconnected Node
layout issue

After renaming several nodes to clean up the canvas, the Sheets node suddenly stayed grayed out and refused to run.

debug_log:

The parameters panel read No input connected. The rename had visually shifted the canvas and silently snapped the wire loose between the AI Agent node and the Sheets node. Reconnecting the two resolved it — a reminder to verify actual wire connections after any layout cleanup.

sheets-node.status
The error panel that revealed the broken node connection.
STEP 05
Gmail Draft Templates & Key-Field Coupling
resolved

When routing candidates, downstream Gmail nodes initially failed to map extracted fields, resulting in empty email drafts.

debug_log:

The AI Agent schema output used keys like Is Fresh Grad: (with trailing colons), but downstream nodes expected variables without colons. I aligned all draft variables and Router parameters to use matching keys (e.g., Name:, Course:, University:, Skills:) and set the Thread ID correctly using {{ $('Gmail Ingestion').item.json.id }}.

02.5 — Validation

End-to-End Integration Testing Journey

To ensure the system runs flawlessly under real-world scenarios, I conducted three distinct test runs using real email scenarios and resume PDF structures.

Test 1: Successful Candidate Ingestion (Dynamic Routing Check)

First, I validated the pipeline's ability to ingest and dynamically route candidates by running two separate test scenarios:

Case A: Junior / Fresh Graduate Route

I submitted a test email containing my own CV: LIPALAM_Benjie_CV.pdf. The keyword filter matched successfully, and Gemini analyzed the profile context. Recognizing the fresh graduate criteria, n8n correctly routed the candidate to the Gmail - Draft Junior Invitation branch.

04 — Roadmap

Known limitations & next steps

This system runs reliably within its current scope, but it isn't "finished" — and I'd rather say so than oversell it. Here's what I'd tackle next for a production environment:

/* backlog — not yet implemented */
☐

Fragile field-key coupling. Every downstream node hard-codes the exact Sheet column names (including trailing colons), so a single header rename can silently break multiple drafts at once — next: a centralized mapping/normalization step right after the AI Agent output, so only one place needs updating if the schema changes.

☐

No failure monitoring. A failed PDF extraction or AI timeout currently fails silently — next: an error branch that alerts Telegram or Slack on any failed run.

☐

Static classification logic. Fresh Grad / Experienced routing uses fixed criteria — next: a configurable rules sheet, reusable across multiple job openings.

☐

No candidate scoring. The system sorts by category, not fit — next: an AI-generated match score (1–10) against job requirements.

☐

Blunt keyword filter. Reduces API cost, but an oddly worded legitimate application could still slip through — next: a more generous fallback rule.
